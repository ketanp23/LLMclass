01 · Counting words

N-gram next-word predictor

An n-gram model predicts the next word using only the last n−1 words, estimated by counting. Change the order and watch coherence rise — until the context has never been seen and the model has nothing to say.

How it works, step by step
  1. Tokenize and count. The training text is split into words, then we tally how often each context is followed by each next word. A bigram stores counts of (previous word → next word); a trigram uses the previous two words as context.
  2. Turn counts into probabilities. The chance of a word given its context is just its share of that context's continuations: P(next | context) = count(context, next) / count(context) These are the bars you see.
  3. The Markov shortcut. Conditioning on the entire sentence history is impossible to estimate, so we assume only the last n−1 words matter. Unigram (n=1) ignores context entirely — pure word frequency, so the output is word salad. Bigram looks back one word; trigram looks back two and reads far more fluently.
  4. Add-k smoothing (the slider). Raw counting assigns probability zero to any pair never seen in training — and one zero makes an entire sentence's probability zero. Add-k fixes this by pretending every word in the vocabulary was seen k extra times after every context: P(next | context) = (count(context, next) + k) / (count(context) + k · V) where V is the vocabulary size. So k = 0 is pure counts (sharpest, but unseen contexts give nothing); k = 1 is "Laplace smoothing," giving every word a floor and flattening the distribution; a small k ≈ 0.1–0.3 keeps rare continuations possible without washing out the real signal.
    Try it: set the order to Trigram, generate until you hit a red "unseen context," then raise the add-k slider — options appear because every word now has a nonzero floor.
  5. Generate text. Sample the next word from the distribution, append it, slide the context window forward by one, and repeat.
  6. Where it breaks. Most possible contexts are never seen (sparsity), the model forgets anything older than n−1 words, and every word is an atomic symbol with no sense of similarity — "cat" and "dog" are as unrelated as "cat" and "the." That last problem is what word embeddings (Lab 02) solve.

02 · Meaning as geometry

Word2Vec & GloVe explorer

Words become vectors positioned so that similar meanings sit close together. That geometry lets you do arithmetic on meaning — and reveals the difference between Word2Vec (predict local context) and GloVe (factorize global co-occurrence counts).

How it works, step by step
  1. Words become vectors. Every word is a point in a multi-dimensional space. In this toy the coordinates are readable features (royalty, gender, tense, animal-type…); real Word2Vec/GloVe learn the coordinates from data, but the geometry behaves the same way.
  2. Similarity is an angle. Two words are "close" when their vectors point the same way, measured by cosine similarity: cos(a, b) = (a · b) / (|a| · |b|) 1 means identical direction, 0 means unrelated. "Nearest neighbours" mode just ranks every word by this score.
  3. Relationships are directions. Because meaning is geometric, consistent relationships become consistent vectors — the step from man→woman is the same direction as king→queen.
  4. Analogy = vector arithmetic. king − man + woman ≈ queen Subtracting man from king isolates the "royalty" direction; adding woman then lands on the royal-and-female point. The dashed parallelogram on the plot is exactly this: the two dashed edges are parallel because the relationship is the same.
  5. The 2D picture (PCA). The vectors have many dimensions, which we can't see. PCA finds the two directions along which the words spread out the most and projects everything onto them — so clusters (royalty, countries, animals, verbs) become visible on a flat screen.
    Hover any point to read its label; the involved words in an analogy or neighbour search are enlarged and colored by cluster.
  6. Word2Vec vs GloVe. Same destination, different training signal. Word2Vec slides a window over text and learns vectors by predicting each word's neighbours (local context). GloVe first builds one global table of how often each pair co-occurs (the heatmap), then learns vectors whose dot product reconstructs the log of those counts. Local prediction vs. global counting — both yield the same useful geometry.

03 · Reading in order

RNN & LSTM lab

A recurrent net folds each new token into a hidden state that carries the past. It works — until the past gets too far away and the gradient vanishes. LSTMs add gates to protect memory. Explore all three below.

How it works, step by step
The recurrence (stepper)
  1. Read one token at a time. The network keeps a hidden state h — a fixed-size vector that summarizes everything seen so far.
  2. Fold each token in. At every step the new state combines the current input with the previous state through the same weights: hₜ = tanh(Wₓ xₜ + Wₕ hₜ₋₁ + b) The bars are those 6 numbers; watch them drift as words arrive. The whole past is squeezed into this one vector — that's the bottleneck attention later removes.
Why the gradient vanishes
  1. Learning long links means reaching back. To connect a word to something many steps earlier, the training error has to travel back through every step in between.
  2. Each step multiplies by ≈ the recurrent weight. So the signal scales like |w|^(distance) If |w| < 1 it shrinks exponentially and the model effectively forgets anything beyond a few steps; if |w| > 1 it explodes instead. The chart shows exactly where the violet RNN curve drops below a usable level.
How LSTM gates fix it
  1. A protected cell state. The LSTM adds a memory line c alongside h, guarded by three gates, each a sigmoid squashing its input to a 0…1 "how much" valve.
  2. Forget, then write. cₜ = fₜ · cₜ₋₁ + iₜ · g̃ₜ The forget gate fₜ decides how much old memory to keep; the input gate iₜ decides how much of the new candidate g̃ₜ to add.
  3. Then expose. hₜ = oₜ · tanh(cₜ) The output gate oₜ decides how much of the cell to reveal as this step's hidden state.
  4. The highway. When forget ≈ 1 and input ≈ 0, the cell passes through unchanged — so gradients flow across many steps without shrinking (the flat cyan line). That's why LSTMs remember far longer than plain RNNs.

04 · Look anywhere

Self-attention visualizer

How it works, step by step
  1. Three roles per word. Each token projects into a Query (what am I looking for?), a Key (what do I offer?), and a Value (what I'll pass along). Real Transformers learn separate projection matrices for these; here we use the embeddings directly so every number is transparent.
  2. Score every pair. For the chosen query word, take the dot product with every key — a higher dot product means the two words are more relevant to each other: score(i, j) = qᵢ · kⱼ
  3. Scale by √dₖ. score / √dₖ Dividing by the square root of the key dimension keeps the numbers from growing huge as dimensions add up, which would push softmax into an all-or-nothing spike. Toggle scaling off to watch the weights sharpen.
  4. Softmax into weights. The scores are turned into a probability distribution that sums to 1 — these are the attention weights (the bars and the heatmap row). softmax(QKᵀ / √dₖ)
  5. Blend the values. The query's output is the weighted sum of every Value: out = Σ weightⱼ · valueⱼ So the new vector for "it" literally becomes a mixture dominated by whatever it attended to — that's how the model resolves what "it" refers to.
    Click it in the tired-animal sentence: most of its attention lands on animal. Switch to the trophy/suitcase sentence and "it" points to a different word.
  6. Causal mask. When generating text left to right, a word must not peek at future words. The mask sets those scores to −∞ so softmax gives them exactly 0 weight — the upper triangle of the heatmap goes dark.

05 · Attention is all you need

Transformer architecture, block by block

A Transformer drops recurrence entirely and stacks attention + feed-forward blocks that run in parallel. Click any block to see what it does and how the tensor shape changes as data flows up the stack.

Click a block to explore

The diagram shows one encoder layer. Data enters at the bottom as token embeddings and flows upward; residual connections (the side arrows) let each block add to the stream rather than replace it.

POSITIONAL ENCODING · hover a position
Each column is a token position · each row a dimension · sinusoids of different frequencies give every position a unique fingerprint
How it works, step by step
  1. Embed the tokens. Each input word becomes a vector of size d_model — the same "words as geometry" idea from Lab 02.
  2. Add positional encoding. Attention treats a sentence as a set, with no inherent order, so we add a position signal to every embedding. Each position gets a unique pattern of sines and cosines at different frequencies (the heatmap). Nearby positions have similar patterns, letting the model reason about distance — now "dog bites man" and "man bites dog" look different.
  3. Multi-head self-attention. Every token attends to every other token (exactly Lab 04), run in several parallel heads that each learn a different kind of relationship. Because it's all matrix multiplication, the whole sequence is processed at once — no step-by-step recurrence.
  4. Add & Norm. A residual connection adds the block's input back to its output, then LayerNorm rescales the result. Residuals mean each layer only learns a small correction, which is what lets very deep stacks train at all.
  5. Feed-forward network. A small MLP is applied to each position independently, usually expanding to a larger width and back. Attention mixes information across tokens; this mixes features within each token.
  6. Stack N identical layers. The output has the same shape as the input, so you can stack these blocks 6, 12, or 96 deep. That depth builds up abstraction and is the backbone of BERT, GPT, and essentially every modern LLM.
